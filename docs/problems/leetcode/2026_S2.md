### 1937. 扣分后的最大得分 

* Score: 2106

* Quality: 0

* Date: 2026/05/05

* [link](https://leetcode.cn/problems/maximum-number-of-points-with-cost/description/)

* Comment:

    這題雖然中間有想到前後綴分解，但是因為基於直覺思考所以沒有完整的想出方法。

    應該要老實地寫出轉移方程，儘管可能是複雜度不佳的轉移，寫下來才能透過一些式子拆分進行優化。

    $$f(r, c) = points(r, c) + max_{k = 0}^{n - 1}{(f(r - 1, k) - |k - c|)}$$

    去掉絕對值可以有兩個式子:

    \[
    \begin{array}{ll}
        f(r, c) = points(r, c) + max_{k = 0}^{n - 1}{(f(r - 1, k) - (c - k))}, \text{ for } k \le c\\
        f(r, c) = points(r, c) + max_{k = 0}^{n - 1}{(f(r - 1, k) - (k - c))}, \text{ for } k < c
    \end{array}
    \]

    DP 常用的優化技巧，把無關項提出來

    \[
    \begin{array}{ll}
        f(r, c) = points(r, c) - c + max_{k = 0}^{n - 1}{(f(r - 1, k) + k)}, \text{ for } k \le c\\
        f(r, c) = points(r, c) + c + max_{k = 0}^{n - 1}{(f(r - 1, k) - k)}, \text{ for } k < c
    \end{array}
    \]

    可以發現 $k \le c$ 和 $k < c$ 分別是前後綴最大，且跟 c 無關，因此利用預處理可以事先算出。

### 3333. 找到初始输入字符串 II

* Score: 2629

* Quality: 0

* Date: 2026/05/28

* [link](https://leetcode.cn/problems/find-the-original-typed-string-ii/description/)

一開始以為是單純的多重背包，結果寫到最後發現不能單純地用普通的多重背包思考。

這樣寫的問題會是複雜度爆炸 $O(mk)$，$m$ 為相鄰不同的字母個數。

接著思考到對於每組大小為 1 的部分可以跳過，且當 $m > k$ 的時候可以直接回傳答案，因為不管怎麼排都會大於 k。

下面這樣寫會通過，但是複雜度還是 $O(mk)$ ，因為 `k -= cnt == 1;` 沒辦法解決根本問題。

```cpp
class Solution {
public:
    const int mod = 1'000'000'000 + 7;
    int possibleStringCount(string s, int k) {
        const int n = s.length();
        if(n < k) return 0;
        vector<int> fq;
        int cnt = 0;
        long long ans = 1;
        for(int i = 0; i < n; ++i) {
            cnt++;
            if(i == n - 1 || s[i] != s[i + 1]) {
                //這是優化關鍵
                if(cnt > 1) {
                    if(k > 0) fq.push_back(cnt);
                    ans = (ans * cnt) % mod;
                }
                k -= cnt == 1;
                cnt = 0;
            }
        }
        //這是優化關鍵
        if(k <= 0) return ans;
        int m = fq.size();
        vector f(2, vector<long long>(k + 1, 0));
        f[0][0] = 1;
        vector<long long> pre(k + 2, 0);
        for(int i = 0; i < m; ++i) {
            int x = fq[i];
            int rest = m - 1 - i;
            for(int j = 0; j <= k; ++j) {
                pre[j + 1] = (pre[j] + f[i & 1][j]) % mod;
            }
            for(int s = k; s >= 0; --s) {
                auto &nxt = f[(i + 1) & 1];
                long long sum = (pre[s - 1 + 1] - pre[max(0, s - x)]) % mod;
                if(s - x < 0) {
                    sum +=  f[i & 1][0] * abs(s - x) % mod;
                }
                sum += mod;
                nxt[s] = sum % mod;
            }
        }
        return f[m & 1][k];
    }
};
```

### 3474. 字典序最小的生成字符串

* Score: 2605

* Quality: 0

* Date: 2026/06/18

* [link](https://leetcode.cn/problems/lexicographically-smallest-generated-string/description/)

這題一開始的想法很簡單先處理完 T 的部分，剩下位置則填入 a ，接著會卡在如何找到一個適合的方法填 F 的部分。

然後貪心的由左至右解決 F 的限制，假設發現遇到的字串跟 str2 相同，那就把最後一個可以更改的位置改成 b。"

首先遇到的問題是當我填了一個沒被 T 限制的字符後，要如何保證不會因此改動到前面已經處理好的 F 限制。

這部分的在 [證明](https://leetcode.cn/problems/lexicographically-smallest-generated-string/solutions/3592764/liang-chong-fang-fa-tan-xin-bao-li-pi-pe-gxyn/)。

除了上面那個問題，那還有一個問題是"為何只需要改成 b?"

```
 i
|  A  |b| B |b|  C  |
     |  A   |b| B |a|  C  |
      k      j
```

先定義:

* s2 = t = A + b + B + a + C
* s1[i] = s1[k] = F
* j 為在處理 s1[i] 時，倒著找到的第一個更可以更改的位置，改成 b。

問題在處理 s[k] = F 時是否會遇到到一個位置 j 此時是倒著遍歷遇到第一個待定的位置，如上圖所示，要因此把 b 改成 c。此時可以得到一個資訊，由 j + 1 位置開始的前綴 B + a + C 會是 t 的前綴。

使用反證法，假設 j 為 s[k] = F 時，倒著遍歷會遇到的第一個待定位置，接著分類討論:

1. 假設 $|A| = |B|$

當 $|A| = |B|$ 如下圖:

```
 i
|  A  |b| B |b|  C  |
        | A |b| B |a|  C  |
      k      j
```

由前面得到的資訊 **B + a + C 會是 t 的前綴**，可以發現 t 的前綴分別可以寫成 $t = A + b ...$ 以及 $t = B + a ...$，又 $|A| = |B|$ 時，會有 $|A + b| = |B + a|$
