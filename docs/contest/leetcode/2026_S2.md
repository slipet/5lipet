Q3 - Minimum Increase to Maximize Special Indices

這題用區間DP的做法，對於位置 i 都有兩種選擇:

假設 $f_i = \{cnt_{i}, op_{i}\}$，$f_i$ 表示前 $i$ 個元素可以產生 $cnt_{i}$ 個峰頂，並且使用 $op_{i}$ 次操作。

1. 不選 i - 1 為峰頂:

    $$f_i = f_{i - 1}= \{cnt_{i - 1}, op_{i - 1}\}$$

2. 選 i - 1 為峰頂:

    $gain(i)$ 是 $i - 1$ 位置為峰頂時產生的貢獻

    $$f_i = \{cnt_{i - 2} + 1, op_{i - 2}+ gain(i)\}$$

在 i 時每次每次都先選能產生更多 $cnt$ 的選擇，然後再選 $op$ 比較小的。

* 這題還可以用狀態機 DP 用另外一個變數 0/1 判斷前面是否有選。

* 接著是靈神的解法 [前後綴分解](https://leetcode.cn/problems/minimum-operations-to-achieve-at-least-k-peaks/solutions/3945005/zuo-fa-lei-si-huan-xing-da-jia-jie-she-p-yu7j/)
* 可以先計算後綴，透過原地撤銷，優化空間複雜度。

```cpp
using ll  = long long;
class Solution {
public:
    long long minIncrease(vector<int>& nums) {
        const int n = nums.size();
        ll pre = 0, suf = 0;
        auto cost = [&](int i) -> int {
            return max(max(nums[i - 1], nums[i + 1]) - nums[i] + 1, 0);
        };
        for (int i = n - 2; i > 0; i -= 2) {
            suf += cost(i);
        }
        if(n & 1) return suf;
        ll res = suf;
        for(int i = 1; i < n - 1; i += 2) {
            pre += cost(i);
            suf -= cost(i + 1);
            res = min(res, suf + pre);
        }
        return res;
    }
};
```

Q4. Good Subsequence Queries

題目要回答每次修改是否能夠滿足題目要求，計算滿足要求的修改。

首先觀察不用修改的情況該如何處理。題目要求選擇一個子序列使得子序列全部元素 $x_i$ 的 $gcd(x_i) = p$ 。

可以觀察到:

1. 只要處理陣列元素為 p 的倍數，全部的 $gcd(x_i) = G$，
2. 選擇越多的元素得到的 G 會越小，所以應該想辦法選全部的 $x_i$，此時全部的元素 $gcd(x_i) = p$，若是選擇的 $x_i$ 數量小於 n 即符合題目要求
3. 接著討論若是選擇 $x_i$ 的個數為 n 的情況，此時最次佳的選擇是去除一個元素 $x_i$ 能否依然構造出題目要求的解 $gcd(x_i')$? $\rightarrow$ 反面思考，什麼情況會有移除一個元素後仍然無法產生 $gcd(x_i')$? ($x_i'$ 表示移除一個 $x_i$ 後剩下的元素 $x_i'$)

    為了便於討論設定 $a_i = \frac{x_i}{p}$，我們的目標是構造出一個集合，移除任何一個元素都會使剩下元素 $gcd(a_i') > 1$。

    1. 首先假設移除第一個元素 $x_1'=1$ 其餘元素為 2，此時序列為 $1,2,2,2,.....$，這時的 $gcd(a_i) = 1, gcd(a_i') = 2$
    2. 接著若是要能夠移除第二個元素 $ａ_2'$，可以將除了第二個元素都乘上 3，會有 $3,2,6,6,.....$，這時的 $gcd(a_i) = 1, gcd(a_i) = 3$
       * 能否再次乘上 2? 假設除第二個元素都乘上 2，會有 $2,2,4,4,.....$，這時 $gcd(a_i) = 2 \rightarrow$ 與我們一開始設定的 $gcd(a_i) = 1$ 的設定不符。
       * 小結論: 每次都只能乘上與前面戶質的元素，也就是說只能乘上質數
    3. 以此類推，為了移除第三個元素乘上 5 ，會有 $15,10,6,30,30,.....$，這時的 $gcd(a_i) = 1, gcd(a_i') = 5$
    
    可以發現按照上面的過程可以不斷的構造出一個序列使得移除第 i 個元素後剩下的元素 $a_i'$ 的 $gcd(a_i') > 1$，但是這樣能夠造的序列長度會因為題目值域限制 $5\times 10^4$，使得我們最多只能使用 $2\times3\times5\times7\times11\times13\times17=510,510 > 5\times 10^4$，可以知道只要 $n > 6$ 時我們無法構造出一個移除一個 $a_i$ 後 $gcd(a_i) > 1$ 的序列 $\rightarrow$　移除一個 $a_i$ 後一定可以得到一個 $gcd(a_i) = 1$ 的序列。

有了上面的結論後可以使用線段樹維護陣列中元素為 $p$ 倍數的 $gcd$。而對於 $n <= 6$ 的情況可以利用前後綴分解的方式找移除一個元素的 $gcd$。

```cpp
for(int i = 0; i < n; ++i) {
    if(gcd(query(0, i - 1), query(i + 1, n - 1)) == target_gcd) {
        return true;
    }
}
```

* 這題的關鍵是 GCD 越多元素得到的 G 越小，嘗試取全部 p 的倍數
* 接著思考是否沒辦法取全部 p 的倍數? 
* 那移除一個 p 的倍數是否能夠符合要求? 什麼情況會不滿足?
* 構造一個序列無法滿足題目要求，不斷的往這邊去構造
* 最後發現 n > 6 一定可以滿足要求
